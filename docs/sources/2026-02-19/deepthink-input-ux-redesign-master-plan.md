# DeepThink 输入体验重构总计划（模板优先 + 双页分工）
> 日期：2026-02-19  
> 目录：`docs/sources/2026-02-19/`

## 1. 目标
1. 降低用户输入门槛，避免“空白输入框 + 随意提问”导致低质量结果。
2. 将业务分析和工程排障分离，减少同页认知负担。
3. 保持现有后端接口不变，优先前端交互重构。

## 2. 现有能力约束
1. `query/stream` 实际必需入参仅 `stock_codes + question`。
2. 后端已具备数据刷新、3个月样本增强、A股市场状态注入。
3. DeepThink 轮次、事件存档、导出、自检能力已存在。

## 3. 实施路径
1. ROUND-UX-1：双页分工（`/deep-think` 业务页、`/deep-think/console` 工程控制台）。
2. ROUND-UX-2：模板优先输入（静态模板 + 槽位）。
3. ROUND-UX-3：输入质量守卫（校验、质量分、建议提示）。
4. ROUND-UX-4：业务聚焦展示（业务页默认只保留分析视角）。
5. ROUND-UX-5：Analysis Studio 联动页重构与乱码文案清理。

## 4. 验收口径
1. 前端构建通过：`npm run build`。
2. 前端类型检查通过：`npx tsc --noEmit`。
3. `/deep-think` 与 `/deep-think/console` 路由可访问，职责分离清晰。
4. 标准分析默认可通过模板生成问题并触发分析。

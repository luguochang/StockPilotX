# 实施路线图：从现有系统到 A2A 互联

日期：2026-02-15

## 1. 阶段化目标
1. 阶段 A（1-2 周）：深度思考接口化
2. 阶段 B（2-3 周）：DeepAgent 执行循环落地
3. 阶段 C（2 周）：A2A 网关与外部代理接入
4. 阶段 D（1-2 周）：评测门禁与治理闭环

## 2. 阶段 A：Deep Think API
- 交付：
  - `/v1/deep-think/*` 接口
  - session/round 数据模型
  - SSE 中间事件
- 验收：
  - 多轮协商可回放
  - 每轮有 agent 证据与分歧分数

## 3. 阶段 B：DeepAgent
- 交付：
  - planner + todo graph
  - subagent 动态派发
  - critic + re-plan
  - budget controller
- 验收：
  - 至少 1 条复杂问题可自动重规划
  - 超预算时可稳定降级

## 4. 阶段 C：A2A 互联
- 交付：
  - A2A gateway
  - agent card registry
  - task lifecycle adapter
- 验收：
  - 与至少 1 个外部代理完成端到端任务
  - 外部失败可自动回退内部链路

## 5. 阶段 D：评测与治理
- 交付：
  - 轨迹评测集
  - 在线抽样质量看板
  - 发布门禁规则
- 验收：
  - 达到既定门槛（引用覆盖、冲突未解率、工具违规率）

## 6. 风险与应对
- 协议变化风险：A2A 适配层隔离，避免核心逻辑耦合。
- 成本激增风险：预算策略与模型路由分层。
- 结果偏差风险：强制反方观点 + critic 审核。

## 7. 最终交付定义（DoD）
- 文档、接口、代码、评测、审计五项齐全。
- 能证明“多 agent 是可解释可验证的决策系统”，不是黑盒问答。
